# MarketEye — Claude 작업 지침

## 응답 규칙
- 코드를 변경할 때마다 **항상 커밋 메시지를 함께 제안**할 것.
- 커밋 메시지는 기존 히스토리 컨벤션을 따른다:
  - 형식: `<type>: <한국어 설명>`
  - type: `feat`(기능 추가), `fix`(버그 수정), `style`(스타일/UI), `refactor`, `docs`, `chore`
  - 예) `fix: 신호 탭 PER/PBR이 한국 종목에서 안 뜨던 문제 수정`

## 프로젝트 개요
- 국내·해외 주식 비교 대시보드. yfinance + KRX 종목 데이터 기반.
- `app.py` — Flask 백엔드 (yfinance 조회, KRX 검색, TTL 캐시, Supabase 동기화)
- `index.html` — 단일 파일 프론트엔드 (Chart.js, 다크 테마)
- `signals.py` — 순수 신호 엔진(차트·펀더멘털·매수존), `infra.py` — 로깅·캐시·yf_call, `providers.py` — 수집층
- `krx_stocks.json` — KRX 종목 폴백 번들

## ⚠️ 배포 회복탄력성 — 코드 변경 시 매번 고려할 것 (반복 사고 예방)
1. **배포하면 서버 인메모리 TTL 캐시가 통째로 비워진다(콜드스타트).** 첫 요청들은 야후
   라이브 호출을 하고, 야후는 데이터센터 IP를 레이트리밋/차단해 느리거나 행(hang)날 수 있다.
   → 어떤 변경도 "콜드스타트 + 야후 무응답"에서 UI가 멈추지 않는지 확인할 것.
2. **프론트는 절대 무한 스피너에 빠지면 안 된다.** 직전 값 로컬 캐시(stale-while-revalidate,
   `_CACHE_KEY`)로 즉시 표시하고, 재시도 소진 시 '다시 시도' 카드로 폴백한다. 이 안전망을
   훼손하지 말 것.
3. **야후 호출은 반드시 `yf_call`(하드 타임아웃·백오프)로 감싼다.** 신호 경로에 무캐시
   야후 호출을 새로 추가하면 콜드스타트 부하가 커진다 — 꼭 필요할 때만, 실패 시 폴백 보장.
4. **검증은 "실제로 렌더되는지"까지 본다.** 특히 `buy_zone`/신호는 호출부 try/except가
   예외를 삼키므로, `None`이 아닌지·값이 실제로 나오는지 단언(assert)할 것. tech_score만
   확인하면 조용히 깨진 걸 놓친다(과거 사고 사례).
5. **`_technical_signal`의 tech_score·매매플랜은 백테스트와 공유·튜닝됨.** 점수 로직 변경 시
   `/api/backtest`로 monotonic·excess 회귀를 확인. 매수존·참고지표는 `with_reasons` 경로
   전용이라 점수·백테스트에 영향 없음(그 불변식을 유지).
